- 题目：旋转图像（leetcode-48）
- 题目描述：给定一个 n × n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。你必须在 原地 旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要 使用另一个矩阵来旋转图像。
- 思路：单纯模拟过程不现实，很自然的想到矩阵转置是一个将行列对换的过程，不难发现，如果先对矩阵进行转置操作，再将每一行进行翻转，就能得到顺时针旋转90度后的矩阵。
- 代码实现：

```python
class Solution:
    def rotate(self, matrix: list[list[int]]) -> None:
        n = len(matrix)
        for i in range(0, n - 1):
            for j in range(i + 1, n):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
        mid = n // 2
        for j in range(mid):
            for i in range(n):
                matrix[i][j], matrix[i][n - 1 - j] = matrix[i][n - 1 - j], matrix[i][j]

```

- 题目：搜索二维矩阵 $\mathrm{II}$（leetcode-240）
- 题目描述：编写一个高效的算法来搜索 $m \times n$ 矩阵 matrix 中的一个目标值 target 。该矩阵具有以下特性：
  - 每行的元素从左到右升序排列。
  - 每列的元素从上到下升序排列。
- 思路：一开始做的时候，没想到怎么做，想从第 1 行和第 1 列分别找到 target 的后驱，然后对于这两行两列围成的小矩阵再搜索，发现这样的思路很奇怪，看了题解发现正确做法是从右上角元素 $matrix[0][n-1]$ 开始搜索，如果 这个元素等于 target，搜索结束；如果这个元素大于 target，那么 target 不可能在这个元素所在的列了，可以将这一列排除掉；如果这个元素小于 target，那么 target 不可能在这个元素所在的行了，可以将这一行排除掉。每次比较都能排除掉一行或者一列。
- 时间复杂度：$O(m+n)$。
- 代码实现：

```python
class Solution:    
    def searchMatrix(self, matrix: list[list[int]], target: int) -> bool:
        m = len(matrix)
        n = len(matrix[0])
        i, j = 0, n - 1

        if matrix[i][j] == target:
            return True

        while 0 <= i < m and 0 <= j < n:
            if matrix[i][j] > target:
                j -= 1
            elif matrix[i][j] < target:
                i += 1
            else:
                return True
        return False
```

- 题目：编辑距离（leetcode-72）
- 题目描述：给你两个单词 word1 和 word2，请你计算出将 word1 转换成 word2 所使用的最少操作数。你可以对一个单词进行如下三种操作：
  - 插入一个字符
  - 删除一个字符
  - 替换一个字符
- 思路：很自然想到是用dp来做，定义 $dp[i][j]$ 表示将 $word1[0:i]$ 转换成 $word2[0:j]$ 的最少操作数。不难发现，如果 $word1[i - 1]$ 和 $word2[j - 1]$ 相等，那么 $dp[i][j]$ 可以通过 $dp[i - 1][j - 1]$ 转移时不用进行额外的操作；如果不相等，那么可以选择插入、删除或者替换操作中的一种，$dp[i][j]$ 从三种状态转移分别对应三种操作，代价分别是 $dp[i][j - 1] + 1$、$dp[i - 1][j] + 1$ 和 $dp[i - 1][j - 1] + 1$，因此 $dp[i][j] = \min(dp[i][j - 1] + 1, dp[i - 1][j] + 1, dp[i - 1][j - 1] + 1)$。最终，dp的转移可以分为如下两种情况：
$$
dp[i][j] = \begin{cases}
\min(dp[i - 1][j - 1], 1 + \min(dp[i][j - 1], dp[i - 1][j])), & if\; word1[i - 1] = word2[j - 1] \\
1 + \min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1]), & if\; word1[i - 1] \neq word2[j - 1]
\end{cases}
$$
- 时间复杂度：$O(mn)$。
- 代码实现：

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m = len(word1)
        n = len(word2)

        if m == 0:
            return n
        elif n == 0:
            return m
        else:
            dp = [[0] * (n + 1) for _ in range(m + 1)]
            for i in range(m + 1):
                dp[i][0] = i 
            for j in range(n + 1):
                dp[0][j] = j
            for i in range(1, m + 1):
                for j in range(1, n + 1):
                    tmp = dp[i - 1][j - 1]
                    if word1[i - 1] != word2[j - 1]:
                        tmp += 1
                    dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1])
                    dp[i][j] = min(dp[i][j], tmp)
            return dp[m][n]
```

- 题目：数字 1 的个数（leetcode-233）
- 题目描述：给定一个整数 n，计算所有小于等于 n 的非负整数中数字 1 出现的个数。
- 思路：上来除了暴力没有想到什么方法，我是区qaq。经过goat指导，发现可以按照数位考虑。计数位 $k$ 表示$1,10,100,1000,\ldots$，我们诸位考虑个位上出现多少个 1，十位上出现多少个 1，百位上出现多少个 1，以此类推。对于每个位 $k$，我们可以将 $n$ 分成三部分：高位 $high = n // (k * 10)$，当前位 $cur = (n // k) \% 10$，低位 $low = n \% k$。如果 $cur == 0$，那么当前位上 1 的数量由高位决定，为 $high * k$；如果 $cur == 1$，那么当前位上 1 的数量由高位和低位共同决定，为 $high * k + low + 1$；如果 $cur > 1$，说明当前位为 1 的这一轮已经完整走完，为 $(high + 1) * k$。我们对每个位进行上述计算，并将结果累加起来即可得到答案。
- 时间复杂度：$O(\log n)$。
- 代码实现：

```python
class Solution:
    def countDigitOne(self, n: int) -> int:
        ans = 0
        k = 1

        while k <= n:
            high = n // (k * 10)
            cur = (n // k) % 10
            low = n % k
            if cur == 0:
                ans += high * k
            elif cur == 1:
                ans += high * k + low + 1
            else:
                ans += (high + 1) * k
            k *= 10
        return ans
```

- 题目：课程表 $IV$ （leetcode-1462）
- 题目描述：你总共需要上 numCourses 门课，课程编号依次为 0 到 numCourses-1 。你会得到一个数组 prerequisite ，其中 prerequisites[i] = $[a_i, b_i]$ 表示如果你想选 $b_i$ 课程，你 必须 先选 $a_i$ 课程。
    - 有的课会有直接的先修课程，比如如果想上课程 1 ，你必须先上课程 0 ，那么会以 [0,1] 数对的形式给出先修课程数对。
    - 先决条件也可以是 间接 的。如果课程 a 是课程 b 的先决条件，课程 b 是课程 c 的先决条件，那么课程 a 就是课程 c 的先决条件。
    - 你也得到一个数组 queries ，其中 queries[j] = $[u_j, v_j]$。对于第 j 个查询，您应该回答课程 $u_j$ 是否是课程 $v_j$ 的先决条件。返回一个布尔数组 answer ，其中 answer[j] 是第 j 个查询的答案。
- 思路：题目数据范围 numCourses 最大为100，考虑直接使用 Floyd 求传递闭包即可。
- 时间复杂度：$O(n^3)$。
- 代码实现：

```python
# Floyd
class Solution:
    def checkIfPrerequisite(self, numCourses: int, prerequisites: list[list[int]], queries: list[list[int]]) -> list[bool]:
        matrix = [[False] * numCourses for _ in range(numCourses)]
        ans = []

        for edge in prerequisites:
            u, v = edge[0], edge[1]
            matrix[u][v] = True

        for k in range(numCourses):
            for i in range(numCourses):
                for j in range(numCourses):
                    matrix[i][j] |= (matrix[i][k] and matrix[k][j])


        for edge in queries:
            begin, end = edge[0], edge[1]
            ans.append(matrix[begin][end])
        return ans

```

```python
# Topological Sort
from collections import deque
    def checkIfPrerequisite(self, numCourses: int, prerequisites: list[list[int]], queries: list[list[int]]) -> list[bool]:
        graph = [[] for _ in range(numCourses)]
        indegree = [0] * numCourses
        pre = [set() for _ in range(numCourses)]

        for u, v in prerequisites:
            graph[u].append(v)
            indegree[v] += 1
        
        dq = deque()
        for _ in range(numCourses):
            if indegree[_] == 0:
                dq.append(_)
        
        while dq:
            u = dq.popleft()
            for v in graph[u]:
                pre[v].append(u)
                pre[v] |= pre[u]
                indegree[v] -= 1
                if indegree[v] == 0:
                    dq.append(v)
        return [u in pre[v] for u, v in queries]
```

```python
# DFS
def checkIfPrerequisite(self, numCourses: int, prerequisites: list[list[int]], queries: list[list[int]]) -> list[bool]:
        graph = [[] for _ in range(numCourses)]
        reach = [[False] * numCourses for _ in range(numCourses)]
        
        for u, v in prerequisites:
            graph[u].append(v)
        
        def dfs(start: int, u: int):
            for v in graph[u]:
                if reach[start][v] == False:
                    reach[start][v] = True
                    dfs(start, v)
        
        for i in range(numCourses):
            dfs(i, i)
        
        return[reach[u][v] for u, v in queries]
```

- 题目：验证二叉搜索树（leetcode-98）
- 题目描述：给你一个二叉树的根节点 root ，判断其是否是一个有效的二叉搜索树。二叉搜索树定义如下：
    - 节点的左子树只包含 严格小于 当前节点的数。
    - 节点的右子树只包含 严格大于 当前节点的数。
    - 所有左子树和右子树自身必须也是二叉搜索树。    
- 思路：这题涉及中序遍历的一个重要性质，即如果一棵树是二叉搜索树，那么它的中序遍历结果一定是一个严格递增的序列。因此，我们可以通过中序遍历来验证这棵树是否满足二叉搜索树的性质。在中序遍历过程中，我们需要记录上一个访问的节点值，并将当前节点值与上一个节点值进行比较。如果当前节点值不大于上一个节点值，那么这棵树就不是二叉搜索树。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
            self.prev = float('-inf')
            def inorder(node: TreeNode) -> bool:
                if not node:
                    return True
                if not inorder(node.left):
                    return False
                if node.val <= self.prev:
                    return False
                self.prev = node.val
                return inorder(node.right)
            return inorder(root)
```

- 题目：零钱兑换（leetcode-322）
- 题目描述：给你一个整数数组 coins ，表示不同面额的硬币；以及一个整数 amount ，表示总金额。计算并返回可以凑成总金额所需的 最少的硬币个数 。如果没有任何一种硬币组合能组成总金额，返回 -1 。你可以认为每种硬币的数量是无限的。
- 思路：这是经典的背包问题，定义$dp[i]$表示凑齐金额$i$所需的最少硬币个数。对于每个金额$i$，我们可以尝试使用每种面额的硬币，如果当前金额$i$大于等于硬币面额$coin$，那么我们可以选择使用这个硬币，此时需要的硬币个数为$dp[i - coin] + 1$。我们需要在所有可用的硬币中选择一个使得$dp[i]$最小的方案，因此转移方程为：$dp[i] = \min(dp[i], dp[i - coin] + 1)$。
- 时间复杂度：$O(n \cdot m)$，其中$n$是金额，$m$是硬币种类数。
- 代码实现：

```python
class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        if amount == 0:
            return 0
        dp = [(amount + 1) for _ in range(amount + 1)]
        dp[0] = 0

        for w in range(1, amount + 1):
            for coin in coins:
                if w >= coin:
                    dp[w] = min(dp[w], dp[w - coin] + 1)
        return dp[amount] if dp[amount] != (amount + 1) else -1
```

- 题目：寻找两个正序数组的中位数（leetcode-4）
- 题目描述：给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。算法的时间复杂度应该为 O(log (m+n)) 。
- 思路：看到题目是和中位数相关，复杂度也有提示是log级别，很自然和二分联系起来。一开始没想到怎么二分，看了提示后恍然大悟，我们可以给两个数组分别划分成两个位置，记划分点分别为i和j，那么如果$i+j=(m+n+1)//2$，就找到了中位数的位置(可以分别从奇偶两种情况验证$m+n+1$是对的)。划分的合法条件是左半部分都$\leq$右半部分，只需要$nums[i - 1] \leq nums[j]$且$nums[j - 1] \leq nums[i]$同时成立。因此，我们只要枚举$i$，就能得到$j$，然后验证划分是否合法，如果不合法，根据条件调整$i$的值即可。最终，如果$m+n$是偶数，那么中位数就是左半部分最大值和右半部分最小值的平均数；如果$m+n$是奇数，那么中位数就是左半部分的最大值。需要注意的是我们总是希望在短数组上二分，这样$j$的计算不容易出现越界。
- 时间复杂度：$O(\log \min(m, n))$。
- 代码实现：

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        l, r = 0, len(nums1)
        count = (len(nums1) + len(nums2) + 1) // 2
        while l <= r:
            i = (l + r) // 2
            j = count - i
            m_left = nums1[i - 1] if i > 0 else float('-inf') 
            m_right = nums1[i] if i < len(nums1) else float('inf')
            n_left = nums2[j - 1] if j > 0 else float('-inf')
            n_right = nums2[j] if j < len(nums2) else float('inf')

            if m_left <= n_right and n_left <= m_right:
                if (len(nums1) + len(nums2)) % 2 == 1:
                    return max(m_left, n_left)
                else:
                    return (max(m_left, n_left) + min(m_right, n_right)) / 2
            elif m_left > n_right:
                r = i - 1
            else:
                l = i + 1
```

- 题目：最长递增子序列的个数（leetcode-673）
- 题目描述：给定一个未排序的整数数组 nums ， 返回最长递增子序列的个数 。注意 这个数列必须是 严格 递增的。
- 思路：自然想到用dp求解。朴素地，我们用$dp[i]$表示以$nums[i]$结尾的最长子序列长度，用$cnt[i]$额外记录对应的最长子序列个数，我们可以得到一套很慢但是能过的代码。
- 时间复杂度：$O(n^2)$（优化前）
- 代码实现：

```python
# 优化前
class Solution:
    def findNumberOfLIS(self, nums: list[int]) -> int:
        length = len(nums)
        dp = [1] * length
        cnt = [1] * length
        ans = 1
        ans_cnt = 0
        for i in range(length):
            for j in range(0, i):
                if nums[i] > nums[j]:
                    if dp[j] + 1 > dp[i]:
                        cnt[i] = cnt[j]
                    elif dp[j] + 1 == dp[i]:
                        cnt[i] += cnt[j]
                    dp[i] = max(dp[i], dp[j] + 1)
                    ans = max(ans, dp[i])
        for i in range(length):
            if dp[i] == ans:
                    ans_cnt += cnt[i]
        return ans_cnt
```

- 题目：相交链表（leetcode-160）
- 题目描述：给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null 。
- 思路：这道题显然利用双指针寻找相交节点。不妨设链表 A 表示为 a + c，链表 B 表示为 b + c，其中 c 是两链表的公共部分。我们可以使用两个指针分别遍历链表 A 和链表 B，当指针到达链表末尾时，重置指针到另一个链表的头部。这样，当两个指针相遇时，如果存在相交节点，那么它们会在相交节点处相遇；如果不存在相交节点，那么 c = null，那么它们会同时走完 a + b 的路程后到达 null。
- 时间复杂度：$O(m + n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> Optional[ListNode]:
        pa, pb = headA, headB
        while pa != pb:
            pa = pa.next if pa else headB
            pb = pb.next if pb else headA
        return pa
```

- 思路：这道题可以使用迭代或者递归的方法来反转
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        prev = None
        cur = head
        while cur:
            nxt = cur.next
            cur.next = prev
            prev = cur
            cur = nxt

        return prev
```

- 题目：回文链表（leetcode-234）
- 题目描述：给你一个单链表的头节点 head ，请你判断该链表是否为回文链表。如果是，返回 true ；否则，返回 false 。
- 思路：这题是一道双指针模板题，可以使用快慢指针先寻找链表中点，然后我们翻转后半段链表，和前半段比较即可。如果链表长度是奇数，我们找中点时跳过它，再往前走一步，因为它不参与比较（eg. 1->1->1->2->1->1->1）。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        if not head or not head.next:
            return True
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        if fast:
            slow = slow.next

        cur = slow
        prev = None
        while cur:
            nxt = cur.next
            cur.next = prev
            prev = cur 
            cur = nxt
        p1 = head
        p2 = prev
        while p2:
            if p2.val != p1.val:
                return False
            p1 = p1.next
            p2 = p2.next
        return True
```

- 题目：环形链表（leetcode-141）
- 题目描述：给你一个链表的头节点 head ，判断链表中是否有环。如果链表中存在环，则返回 true ；否则，返回 false 。
- 思路：用快慢指针来判断链表是否有环。快指针每次走两步，慢指针每次走一步，如果链表中存在环，那么快指针和慢指针最终会相遇；如果链表中不存在环，那么快指针会先到达链表末尾。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

            if slow == fast:
                return True
        return False
```

- 题目：环形链表 II（leetcode-142）
- 题目描述：给定一个链表的头节点 head ，返回链表开始入环的第一个节点。如果链表无环，则返回 null 。
- 思路：这题在判断链表是否有环的基础上，增加了寻找入环节点的步骤。首先，我们使用快慢指针来判断链表是否有环。如果链表中存在环，那么快指针和慢指针最终会相遇。此时，我们将其中一个指针重新指向链表的头节点，然后两个指针每次都走一步。当它们再次相遇时，相遇的节点就是链表开始入环的第一个节点。证明：设链表头节点到入环节点的距离为a，入环节点到相遇节点的距离为b，相遇节点到入环节点的距离为c，那么快指针走过的路程是慢指针的两倍，即2(a + b) = a + b + k * (b + c)，其中k是快指针在环内绕行的圈数。化简后得到a = (k - 1) * (b + c) + c，此时$a \equiv c \mod (b + c)$，因此当我们将其中一个指针重新指向链表的头节点，并且两个指针每次都走一步时，它们会在入环节点相遇。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

            if slow == fast:
                break
        if not fast or not fast.next:
            return None
        else:
            p = head
            while p != slow:
                p = p.next
                slow = slow.next
            return p
```

```python
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def swapPairs(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        a = head
        prev = dummy
        while prev.next and prev.next.next:
            b = a.next
            nxt = b.next
            prev.next = b
            b.next = a
            a.next = nxt
            prev = a
            a = nxt
        return dummy.next
```

```python
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        group_prev = dummy
        while True:
            kth = group_prev
            for _ in range(k):
                kth = kth.next
                if not kth:
                    return dummy.next
            group_next = kth.next
            cur = group_prev.next
            prev = group_next
            tmp = None
            while cur != group_next:
                tmp = cur.next
                cur.next = prev
                prev = cur
                cur = tmp
            tmp = group_prev.next
            group_prev.next = kth
            group_prev = tmp
        return dummy.next
```

- 思路：这题是归并排序的链表版本，我们要做的事情有两件：对于一个链表，要找到中点，这个在之前的题目有用过快慢指针实现；对于两个有序链表，要将它们合并成一个有序链表，这个在之前的题目也有用过双指针实现。我们可以使用递归来实现归并排序，首先找到链表的中点，将链表分成两半，然后递归地对两半进行排序，最后将排序好的两半合并成一个有序链表。

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def merge(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> ListNode:
        dummy = ListNode()
        cur = dummy
        while l1 and l2:
            if l1.val < l2.val:
                cur.next = l1
                l1 = l1.next
            else:
                cur.next = l2
                l2 = l2.next
            cur = cur.next
        cur.next = l1 if l1 else l2
        return dummy.next


    def sortList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        if not head or not head.next:
            return head
        slow, fast = head, head.next
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        mid = slow.next
        slow.next = None
        left = self.sortList(head)
        right = self.sortList(mid)

        return self.merge(left, right)
```

- 题目：LRU 缓存（leetcode-146）
- 题目描述：请你设计并实现一个满足  LRU (最近最少使用) 缓存 约束的数据结构。实现 LRUCache 类：
    - LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
    - int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
    - void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字。
    - 函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。                               
- 思路：这题是一道模拟题，我们很自然的想用哈希表来表示key-value对应关系，这样get操作就能在O(1)时间内完成。对于put操作，我们需要在插入新元素时判断是否超过容量，如果超过了，我们需要删除最近最少使用的元素，我们知道链表的删除操作是O(1)的，因此我们可以使用一个双向链表来维护元素的使用顺序。每当我们访问一个元素时，我们就将它移动到链表的头部，这样链表的尾部就会始终保持最近最少使用的元素。当我们需要删除最近最少使用的元素时，我们只需要删除链表尾部的节点即可。为了实现这个功能，我们需要在哈希表中存储每个key对应的链表节点，这样我们就能在O(1)时间内找到要移动或删除的节点。需要注意的是，
- 代码实现：

```python
from typing import Deque
class Node:
    def __init__(self, key = 0, val=0, prev=None, next=None):
        self.key = key
        self.val = val
        self.prev = prev
        self.next = next

class LRUCache:

    def __init__(self, capacity: int):
        self.capacity = capacity
        self.kv = dict()
        self.head = Node()
        self.tail = Node()
        self.head.next = self.tail
        self.tail.prev = self.head


    def get(self, key: int) -> int:
        if key not in self.kv:
            return -1
        node = self.kv[key]
        node.prev.next = node.next
        node.next.prev = node.prev
        node.prev = self.head
        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node
        return node.val
        

    def put(self, key: int, value: int) -> None:
        if key in self.kv:
            self.kv[key].val = value
            node = self.kv[key]
            node.prev.next = node.next
            node.next.prev = node.prev
            node.prev = self.head
            node.next = self.head.next
            self.head.next.prev = node
            self.head.next = node
        else:
            if len(self.kv) == self.capacity:
                rm_node = self.tail.prev
                rm_key = rm_node.key
                self.kv.pop(rm_key, None)
                self.tail.prev = rm_node.prev
                rm_node.prev.next = self.tail
            new_node = Node(key, value)
            self.kv[key] = new_node
            new_node.next = self.head.next
            new_node.prev = self.head
            self.head.next.prev = new_node
            self.head.next = new_node
```

- 题目：岛屿数量（leetcode-200）
给你一个由 '1'（陆地）和 '0'（水）组成的的二维网格，请你计算网格中岛屿的数量。岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。此外，你可以假设该网格的四条边均被水包围。

```python
    def numIslands(self, grid: list[list[str]]) -> int:
        n = len(grid)
        m = len(grid[0])
        vis = [[0 for _ in range(m)] for _ in range(n)]

        def dfs(i, j, n, m):
            vis[i][j] = 1

                dfs(i - 1, j, n, m)
            if i + 1 < n and vis[i + 1][j] == 0 and grid[i + 1][j] == "1":
                dfs(i + 1, j, n, m)
            if j - 1 >= 0 and vis[i][j - 1] == 0 and grid[i][j - 1] == "1":
                dfs(i, j - 1, n, m)
            if j + 1 < m and vis[i][j + 1] == 0 and grid[i][j + 1] == "1":
                dfs(i, j + 1, n, m)

        for i in range(n):
            for j in range(m):
                if grid[i][j] == "1" and vis[i][j] == 0:
                    ans += 1
                    dfs(i, j, n, m)
        return ans
```

- 腐烂的橘子（leetcode-994）
- 题目描述：在给定的网格中，每个单元格可以有以下三个值之一：
    - 值 0 代表空单元格；
    - 值 1 代表新鲜橘子；
    - 值 2 代表腐烂的橘子。

    每分钟，任何与腐烂的橘子（在 4 个正方向上）相邻的新鲜橘子都会腐烂。返回直到单元格中没有新鲜橘子为止所必须经过的最小分钟数。如果不可能，返回 -1 。
- 思路：这题是经典的BFS题，我们可以将所有腐烂的橘子作为BFS的起点，然后每次扩散到相邻的新鲜橘子，并将它们加入队列中，直到队列为空。我们需要维护一个计数器来记录分钟数，每次扩散完一轮后，分钟数加1。最后，我们需要检查是否还有新鲜橘子，如果有，说明无法完全腐烂，返回-1；如果没有，返回分钟数。
- 时间复杂度：$O(m \cdot n)$。
- 代码实现：
```python
from typing import Deque

class Solution:
    def orangesRotting(self, grid: list[list[int]]) -> int:
        cnt = 0
        time = 0
        n = len(grid)
        m = len(grid[0])
        dq = Deque()
        for i in range(n):
            for j in range(m):
                if grid[i][j] == 1:
                    cnt += 1
                elif grid[i][j] == 2:
                    dq.append((i, j, 0))
        if cnt == 0:
            return 0
        while dq:
            if cnt == 0:
                return time
            t = dq.popleft()
            i, j, cur = t
            if i - 1 >= 0 and grid[i - 1][j] == 1:
                grid[i - 1][j] = 2
                cnt -= 1
                dq.append((i - 1, j, cur + 1))
            if i + 1 < n and grid[i + 1][j] == 1:
                grid[i + 1][j] = 2
                cnt -= 1
                dq.append((i + 1, j, cur + 1))
            if j - 1 >= 0 and grid[i][j - 1] == 1:
                grid[i][j - 1] = 2
                cnt -= 1
                dq.append((i, j - 1, cur + 1))
            if j + 1 < m and grid[i][j + 1] == 1:
                grid[i][j + 1] = 2
                cnt -= 1
                dq.append((i, j + 1, cur + 1))
            time = max(cur + 1, time)
        return -1
```

- 题目：课程表（leetcode-207）
- 题目描述：你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses-1 。在选修某些课程之前需要一些先修课程。先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，表示如果要学习课程 ai 则 必须 先学习课程 bi 。请你判断是否可能完成所有课程的学习？如果可以，返回 true ；否则，返回 false 。
- 思路：这是一道拓扑排序模板题。我们可以将课程和先修课程的关系看作一个有向图，其中每个课程是一个节点，每个先修关系是一个有向边。我们需要判断这个有向图是否存在环，如果存在环，说明无法完成所有课程的学习；如果不存在环，说明可以完成所有课程的学习。我们可以使用Kahn算法来实现拓扑排序，首先计算每个节点的入度，然后将入度为0的节点加入队列中，依次处理队列中的节点，将它们从图中删除，并更新相邻节点的入度，如果相邻节点的入度变为0，则将其加入队列中。最后，如果处理完所有节点后还有节点未被处理，说明存在环，返回false；否则，返回true。
- 时间复杂度：$O(V + E)$，其中V是课程的数量，E是先修关系的数量。
- 代码实现：
```python
from typing import Deque

class Solution:
    def canFinish(self, numCourses: int, prerequisites: list[list[int]]) -> bool:
        if not prerequisites:
            return True
        n = len(prerequisites)
        edges = [[] for _ in range(numCourses)]
        ans = []

        dq = Deque()
        indegree = [0 for _ in range(numCourses)]
        for i in range(n):
            u, v = prerequisites[i]
            indegree[u] += 1
            edges[v].append(u)
        for idx, cnt in enumerate(indegree):
            if cnt == 0:
                dq.append(idx)
        while dq:
            u = dq.popleft()
            ans.append(u)
            for v in edges[u]:
                indegree[v] -= 1
                if indegree[v] == 0:
                    dq.append(v)
        if len(ans) != numCourses:
            return False
        return True
```

- 题目：实现 Trie (前缀树)（leetcode-208）
- 题目描述：Trie（发音类似 "try"）或者说 前缀树 是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这个数据结构有相当多的应用情景，例如自动补全和拼写检查。请你实现 Trie 类：
    - Trie() 初始化前缀树对象。
    - void insert(String word) 向前缀树中插入字符串 word 。
    - boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
    - boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀 prefix 在前缀树中，返回 true ；否则，返回 false 。
- 思路：Trie树的每个节点包含一个长度为26的数组children，表示26个字母对应的子节点，以及一个布尔变量is_end，表示是否有单词以当前节点结尾。插入操作时，我们从根节点开始，依次处理单词中的每个字符，如果对应的子节点不存在，就创建一个新的节点；最后将最后一个节点的is_end设置为True。搜索操作时，我们同样从根节点开始，依次处理单词中的每个字符，如果对应的子节点不存在，说明单词不在Trie树中；如果最后一个字符对应的节点存在且is_end为True，说明单词在Trie树中。前缀搜索操作时，我们从根节点开始，依次处理前缀中的每个字符，如果对应的子节点不存在，说明没有单词以该前缀开头；如果前缀中的所有字符都能找到对应的子节点，说明至少有一个单词以该前缀开头。
- 时间复杂度：插入、搜索和前缀搜索的时间复杂度都是$O(m)$，其中m是单词的长度。
- 代码实现：
```python
class Trie:

    def __init__(self):
        self.root = {}

    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node:
                node[ch] = {}
            node = node[ch]
        node["#"] = True

    def search(self, word: str) -> bool:
        node = self.root
        for ch in word:
            if ch not in node:
                return False
            node = node[ch]
        return "#" in node

    def startsWith(self, prefix: str) -> bool:
        node = self.root
        for ch in prefix:
            if ch not in node:
                return False
            node = node[ch]
        return True
```

- 题目：买卖股票的最佳时机（leetcode-121）
- 题目描述：给定一个数组 prices ，其中 prices[i] 是一支给定股票第 i 天的价格。你只能选择某一天买入这只股票，并选择在未来的某一个不同的日子卖出该股票。设计一个算法来计算你所能获取的最大利润。返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。
- 思路：这题是经典的动态规划题，我们记 dp[i] 表示到第 i 天的最低价格，然后遍历价格数组，对于每个价格，我们计算当前价格与 dp[i] 的差值，如果这个差值大于当前的最大利润 ans，我们就更新 ans。最后返回 ans 即可。
- 时间复杂度：$O(n)$。
- 代码实现：
```python
class Solution:
    def maxProfit(self, prices: list[int]) -> int:
        ans = 0
        dp = [0 for _ in range(len(prices))]
        for i, pri in enumerate(prices):
            if i == 0:
                dp[i] = pri
            else:
                dp[i] = min(dp[i -1], prices[i - 1])
            ans = max(ans, pri - dp[i])
        return ans
```

- 思路：这题很显然是贪心，可以维护当前能跳到的最远的位置，当遍历完所有能跳到的位置后，如果最远位置大于等于最后一个位置的索引，说明可以到达最后一个位置；否则，说明无法到达最后一个位置。
- 时间复杂度：$O(n)$。
- 代码实现：
```python
class Solution:
    def canJump(self, nums: list[int]) -> bool:
        end = 0
        for i in range(len(nums)):
            if i <= end:
                end = max(end, i + nums[i])
            if end >= len(nums) - 1:
                return True
        return False
```

- 题目描述：给定一个非负整数数组 nums ，你最初位于数组的第一个位置。数组中的每个元素代表你在该位置可以跳跃的最大长度。你的目标是使用最少的跳跃次数到达数组的最后一个位置。假设你总是可以到达数组的最后一个位置。
- 思路：这题也是贪心的思路，我们维护当前能跳到的最远边界，只要接下来遍历的位置没超过当前边界，那么跳跃次数就不增加；当遍历的位置超过当前边界时，说明需要增加一次跳跃次数，并且更新边界为当前能跳到的最远位置。最后返回跳跃次数即可。
- 时间复杂度：$O(n)$。
- 代码实现：
```python
class Solution:
    def jump(self, nums: list[int]) -> int:
        jumps = 0
        cur_end = 0
        farthest = 0

        for i in range(len(nums) - 1):
            farthest = max(farthest, i + nums[i])

            if i == cur_end:
                jumps += 1
                cur_end = farthest

        return jumps
```

- 题目描述：字符串 S 由小写字母组成。我们要把这个字符串划分为尽可能多的片段，同一字母最多出现在一个片段中。返回一个表示每个字符串片段的长度的列表。
- 思路：这题的思路是先遍历字符串，记录每个字母最后一次出现的位置。然后我们再次遍历字符串，维护一个当前片段的结束位置end，每当我们遇到一个字母时，我们就更新end为该字母最后一次出现的位置。我们还需要维护一个当前片段的起始位置start，当我们遍历到当前位置i等于end时，说明当前片段已经结束，我们就将当前片段的长度添加到结果列表中，并将start更新为i+1，开始下一个片段的划分。
- 时间复杂度：$O(n)$。
- 代码实现：
```python
class Solution:
    def partitionLabels(self, s: str) -> list[int]:
        last = {ch: i for i, ch in enumerate(s)}
        ans = []
        start = 0
        end = 0

        for i, ch in enumerate(s):
            end = max(end, last[ch])

            if i == end:
                ans.append(end - start + 1)
                start = i + 1

        return ans
```

- 题目：数组中的第K个最大元素（leetcode-215）
- 题目描述：在未排序的数组中找到第 k 个最大的元素。请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。你必须设计并实现时间复杂度为 O(n) 的算法解决此问题。
- 思路：这题可以使用快速选择算法来实现，快速选择算法是快速排序算法的一个应用，它的平均时间复杂度为O(n)。我们将寻找第 K 大转化为寻找第 N-K 小，其中 N 是数组的长度。我们选择一个基准元素，将数组分成两部分，一部分是小于基准元素的，另一部分是大于基准元素的。如果基准元素的位置正好是 N-K，那么我们就找到了第 K 大的元素；如果基准元素的位置小于 N-K，那么我们需要在右半部分继续寻找；如果基准元素的位置大于 N-K，那么我们需要在左半部分继续寻找。我们不断重复这个过程，直到找到第 K 大的元素。
- 时间复杂度：平均时间复杂度为$O(n)$，最坏情况下为$O(n^2)$。
- 代码实现：
```python
import random

class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        target = len(nums) - k
        l, r = 0, len(nums) - 1

        while l <= r:
            pivot = nums[random.randint(l, r)]

            lt, i, gt = l, l, r

            while i <= gt:
                if nums[i] < pivot:
                    nums[lt], nums[i] = nums[i], nums[lt]
                    lt += 1
                    i += 1
                elif nums[i] > pivot:
                    nums[i], nums[gt] = nums[gt], nums[i]
                    gt -= 1
                else:
                    i += 1

            if target < lt:
                r = lt - 1
            elif target > gt:
                l = gt + 1
            else:
                return nums[target]
```

- 题目：前 K 个高频元素（leetcode-347）
- 题目描述：给你一个整数数组 nums 和一个整数 k ，请你返回其中出现频率前 k 高的元素。你可以按 任意顺序 返回答案。
- 思路：这题可以使用桶排序来实现，首先我们统计每个元素出现的频率，然后我们创建一个桶数组，桶的索引表示频率，桶中的元素是具有该频率的数字。最后我们从桶的末尾开始遍历，直到找到前 K 个高频元素。
- 时间复杂度：$O(n)$。
- 代码实现：
```python
from collections import Counter

class Solution:
    def topKFrequent(self, nums: list[int], k: int) -> list[int]:
        freq = Counter(nums)
        buk = [[] for _ in range(len(nums) + 1)]

        for num, count in freq.items():
            buk[count].append(num)
        ans = []
        for count in range(len(buk) - 1, 0, -1):
            for num in buk[count]:
                ans.append(num)
                if len(ans) == k:
                    return ans
        return ans
```

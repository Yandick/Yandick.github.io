- 题目：旋转图像（leetcode-48）
- 题目描述：给定一个 n × n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。你必须在 原地 旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要 使用另一个矩阵来旋转图像。
- 思路：单纯模拟过程不现实，很自然的想到矩阵转置是一个将行列对换的过程，不难发现，如果先对矩阵进行转置操作，再将每一行进行翻转，就能得到顺时针旋转90度后的矩阵。
- 代码实现：

```python
class Solution:
    def rotate(self, matrix: list[list[int]]) -> None:
        n = len(matrix)
        for i in range(0, n - 1):
            for j in range(i + 1, n):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
        mid = n // 2
        for j in range(mid):
            for i in range(n):
                matrix[i][j], matrix[i][n - 1 - j] = matrix[i][n - 1 - j], matrix[i][j]

```

- 题目：搜索二维矩阵 $\mathrm{II}$（leetcode-240）
- 题目描述：编写一个高效的算法来搜索 $m \times n$ 矩阵 matrix 中的一个目标值 target 。该矩阵具有以下特性：
  - 每行的元素从左到右升序排列。
  - 每列的元素从上到下升序排列。
- 思路：一开始做的时候，没想到怎么做，想从第 1 行和第 1 列分别找到 target 的后驱，然后对于这两行两列围成的小矩阵再搜索，发现这样的思路很奇怪，看了题解发现正确做法是从右上角元素 $matrix[0][n-1]$ 开始搜索，如果 这个元素等于 target，搜索结束；如果这个元素大于 target，那么 target 不可能在这个元素所在的列了，可以将这一列排除掉；如果这个元素小于 target，那么 target 不可能在这个元素所在的行了，可以将这一行排除掉。每次比较都能排除掉一行或者一列。
- 时间复杂度：$O(m+n)$。
- 代码实现：

```python
class Solution:    
    def searchMatrix(self, matrix: list[list[int]], target: int) -> bool:
        m = len(matrix)
        n = len(matrix[0])
        i, j = 0, n - 1

        if matrix[i][j] == target:
            return True

        while 0 <= i < m and 0 <= j < n:
            if matrix[i][j] > target:
                j -= 1
            elif matrix[i][j] < target:
                i += 1
            else:
                return True
        return False
```

- 题目：编辑距离（leetcode-72）
- 题目描述：给你两个单词 word1 和 word2，请你计算出将 word1 转换成 word2 所使用的最少操作数。你可以对一个单词进行如下三种操作：
  - 插入一个字符
  - 删除一个字符
  - 替换一个字符
- 思路：很自然想到是用dp来做，定义 $dp[i][j]$ 表示将 $word1[0:i]$ 转换成 $word2[0:j]$ 的最少操作数。不难发现，如果 $word1[i - 1]$ 和 $word2[j - 1]$ 相等，那么 $dp[i][j]$ 可以通过 $dp[i - 1][j - 1]$ 转移时不用进行额外的操作；如果不相等，那么可以选择插入、删除或者替换操作中的一种，$dp[i][j]$ 从三种状态转移分别对应三种操作，代价分别是 $dp[i][j - 1] + 1$、$dp[i - 1][j] + 1$ 和 $dp[i - 1][j - 1] + 1$，因此 $dp[i][j] = \min(dp[i][j - 1] + 1, dp[i - 1][j] + 1, dp[i - 1][j - 1] + 1)$。最终，dp的转移可以分为如下两种情况：
$$
dp[i][j] = \begin{cases}
\min(dp[i - 1][j - 1], 1 + \min(dp[i][j - 1], dp[i - 1][j])), & if\; word1[i - 1] = word2[j - 1] \\
1 + \min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1]), & if\; word1[i - 1] \neq word2[j - 1]
\end{cases}
$$
- 时间复杂度：$O(mn)$。
- 代码实现：

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m = len(word1)
        n = len(word2)

        if m == 0:
            return n
        elif n == 0:
            return m
        else:
            dp = [[0] * (n + 1) for _ in range(m + 1)]
            for i in range(m + 1):
                dp[i][0] = i 
            for j in range(n + 1):
                dp[0][j] = j
            for i in range(1, m + 1):
                for j in range(1, n + 1):
                    tmp = dp[i - 1][j - 1]
                    if word1[i - 1] != word2[j - 1]:
                        tmp += 1
                    dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1])
                    dp[i][j] = min(dp[i][j], tmp)
            return dp[m][n]
```

- 题目：数字 1 的个数（leetcode-233）
- 题目描述：给定一个整数 n，计算所有小于等于 n 的非负整数中数字 1 出现的个数。
- 思路：上来除了暴力没有想到什么方法，我是区qaq。经过goat指导，发现可以按照数位考虑。计数位 $k$ 表示$1,10,100,1000,\ldots$，我们诸位考虑个位上出现多少个 1，十位上出现多少个 1，百位上出现多少个 1，以此类推。对于每个位 $k$，我们可以将 $n$ 分成三部分：高位 $high = n // (k * 10)$，当前位 $cur = (n // k) \% 10$，低位 $low = n \% k$。如果 $cur == 0$，那么当前位上 1 的数量由高位决定，为 $high * k$；如果 $cur == 1$，那么当前位上 1 的数量由高位和低位共同决定，为 $high * k + low + 1$；如果 $cur > 1$，说明当前位为 1 的这一轮已经完整走完，为 $(high + 1) * k$。我们对每个位进行上述计算，并将结果累加起来即可得到答案。
- 时间复杂度：$O(\log n)$。
- 代码实现：

```python
class Solution:
    def countDigitOne(self, n: int) -> int:
        ans = 0
        k = 1

        while k <= n:
            high = n // (k * 10)
            cur = (n // k) % 10
            low = n % k
            if cur == 0:
                ans += high * k
            elif cur == 1:
                ans += high * k + low + 1
            else:
                ans += (high + 1) * k
            k *= 10
        return ans
```

- 题目：课程表 $IV$ （leetcode-1462）
- 题目描述：你总共需要上 numCourses 门课，课程编号依次为 0 到 numCourses-1 。你会得到一个数组 prerequisite ，其中 prerequisites[i] = $[a_i, b_i]$ 表示如果你想选 $b_i$ 课程，你 必须 先选 $a_i$ 课程。
    - 有的课会有直接的先修课程，比如如果想上课程 1 ，你必须先上课程 0 ，那么会以 [0,1] 数对的形式给出先修课程数对。
    - 先决条件也可以是 间接 的。如果课程 a 是课程 b 的先决条件，课程 b 是课程 c 的先决条件，那么课程 a 就是课程 c 的先决条件。
    - 你也得到一个数组 queries ，其中 queries[j] = $[u_j, v_j]$。对于第 j 个查询，您应该回答课程 $u_j$ 是否是课程 $v_j$ 的先决条件。返回一个布尔数组 answer ，其中 answer[j] 是第 j 个查询的答案。
- 思路：题目数据范围 numCourses 最大为100，考虑直接使用 Floyd 求传递闭包即可。
- 时间复杂度：$O(n^3)$。
- 代码实现：

```python
# Floyd
class Solution:
    def checkIfPrerequisite(self, numCourses: int, prerequisites: list[list[int]], queries: list[list[int]]) -> list[bool]:
        matrix = [[False] * numCourses for _ in range(numCourses)]
        ans = []

        for edge in prerequisites:
            u, v = edge[0], edge[1]
            matrix[u][v] = True

        for k in range(numCourses):
            for i in range(numCourses):
                for j in range(numCourses):
                    matrix[i][j] |= (matrix[i][k] and matrix[k][j])


        for edge in queries:
            begin, end = edge[0], edge[1]
            ans.append(matrix[begin][end])
        return ans

```

# Topological Sort
from collections import deque
    def checkIfPrerequisite(self, numCourses: int, prerequisites: list[list[int]], queries: list[list[int]]) -> list[bool]:
        graph = [[] for _ in range(numCourses)]
        indegree = [0] * numCourses
        pre = [set() for _ in range(numCourses)]

        for u, v in prerequisites:
            graph[u].append(v)
            indegree[v] += 1
        
        dq = deque()
        for _ in range(numCourses):
            if indegree[_] == 0:
                dq.append(_)
        
        while dq:
            u = dq.popleft()
            for v in graph[u]:
                pre[v].append(u)
                pre[v] |= pre[u]
                indegree[v] -= 1
                if indegree[v] == 0:
                    dq.append(v)
        return [u in pre[v] for u, v in queries]
```

# DFS
def checkIfPrerequisite(self, numCourses: int, prerequisites: list[list[int]], queries: list[list[int]]) -> list[bool]:
        graph = [[] for _ in range(numCourses)]
        reach = [[False] * numCourses for _ in range(numCourses)]
        
        for u, v in prerequisites:
            graph[u].append(v)
        
        def dfs(start: int, u: int):
            for v in graph[u]:
                if reach[start][v] == False:
                    reach[start][v] = True
                    dfs(start, v)
        
        for i in range(numCourses):
            dfs(i, i)
        
        return[reach[u][v] for u, v in queries]
```

- 题目：验证二叉搜索树（leetcode-98）
- 题目描述：给你一个二叉树的根节点 root ，判断其是否是一个有效的二叉搜索树。二叉搜索树定义如下：
    - 节点的左子树只包含 严格小于 当前节点的数。
    - 节点的右子树只包含 严格大于 当前节点的数。
    - 所有左子树和右子树自身必须也是二叉搜索树。    
- 思路：这题涉及中序遍历的一个重要性质，即如果一棵树是二叉搜索树，那么它的中序遍历结果一定是一个严格递增的序列。因此，我们可以通过中序遍历来验证这棵树是否满足二叉搜索树的性质。在中序遍历过程中，我们需要记录上一个访问的节点值，并将当前节点值与上一个节点值进行比较。如果当前节点值不大于上一个节点值，那么这棵树就不是二叉搜索树。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right
class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
            self.prev = float('-inf')
            def inorder(node: TreeNode) -> bool:
                if not node:
                    return True
                if not inorder(node.left):
                    return False
                if node.val <= self.prev:
                    return False
                self.prev = node.val
                return inorder(node.right)
            return inorder(root)
```

- 题目：零钱兑换（leetcode-322）
- 题目描述：给你一个整数数组 coins ，表示不同面额的硬币；以及一个整数 amount ，表示总金额。计算并返回可以凑成总金额所需的 最少的硬币个数 。如果没有任何一种硬币组合能组成总金额，返回 -1 。你可以认为每种硬币的数量是无限的。
- 思路：这是经典的背包问题，定义$dp[i]$表示凑齐金额$i$所需的最少硬币个数。对于每个金额$i$，我们可以尝试使用每种面额的硬币，如果当前金额$i$大于等于硬币面额$coin$，那么我们可以选择使用这个硬币，此时需要的硬币个数为$dp[i - coin] + 1$。我们需要在所有可用的硬币中选择一个使得$dp[i]$最小的方案，因此转移方程为：$dp[i] = \min(dp[i], dp[i - coin] + 1)$。
- 时间复杂度：$O(n \cdot m)$，其中$n$是金额，$m$是硬币种类数。
- 代码实现：

```python
class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        if amount == 0:
            return 0
        dp = [(amount + 1) for _ in range(amount + 1)]
        dp[0] = 0

        for w in range(1, amount + 1):
            for coin in coins:
                if w >= coin:
                    dp[w] = min(dp[w], dp[w - coin] + 1)
        return dp[amount] if dp[amount] != (amount + 1) else -1
```

- 题目：寻找两个正序数组的中位数（leetcode-4）
- 题目描述：给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。算法的时间复杂度应该为 O(log (m+n)) 。
- 思路：看到题目是和中位数相关，复杂度也有提示是log级别，很自然和二分联系起来。一开始没想到怎么二分，看了提示后恍然大悟，我们可以给两个数组分别划分成两个位置，记划分点分别为i和j，那么如果$i+j=(m+n+1)//2$，就找到了中位数的位置(可以分别从奇偶两种情况验证$m+n+1$是对的)。划分的合法条件是左半部分都$\leq$右半部分，只需要$nums[i - 1] \leq nums[j]$且$nums[j - 1] \leq nums[i]$同时成立。因此，我们只要枚举$i$，就能得到$j$，然后验证划分是否合法，如果不合法，根据条件调整$i$的值即可。最终，如果$m+n$是偶数，那么中位数就是左半部分最大值和右半部分最小值的平均数；如果$m+n$是奇数，那么中位数就是左半部分的最大值。需要注意的是我们总是希望在短数组上二分，这样$j$的计算不容易出现越界。
- 时间复杂度：$O(\log \min(m, n))$。
- 代码实现：

```python
class Solution:
    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1
        l, r = 0, len(nums1)
        count = (len(nums1) + len(nums2) + 1) // 2
        while l <= r:
            i = (l + r) // 2
            j = count - i
            m_left = nums1[i - 1] if i > 0 else float('-inf') 
            m_right = nums1[i] if i < len(nums1) else float('inf')
            n_left = nums2[j - 1] if j > 0 else float('-inf')
            n_right = nums2[j] if j < len(nums2) else float('inf')

            if m_left <= n_right and n_left <= m_right:
                if (len(nums1) + len(nums2)) % 2 == 1:
                    return max(m_left, n_left)
                else:
                    return (max(m_left, n_left) + min(m_right, n_right)) / 2
            elif m_left > n_right:
                r = i - 1
            else:
                l = i + 1
```

- 题目：最长递增子序列的个数（leetcode-673）
- 题目描述：给定一个未排序的整数数组 nums ， 返回最长递增子序列的个数 。注意 这个数列必须是 严格 递增的。
- 思路：自然想到用dp求解。朴素地，我们用$dp[i]$表示以$nums[i]$结尾的最长子序列长度，用$cnt[i]$额外记录对应的最长子序列个数，我们可以得到一套很慢但是能过的代码。
- 时间复杂度：$O(n^2)$（优化前）
- 代码实现：

```python
# 优化前
class Solution:
    def findNumberOfLIS(self, nums: list[int]) -> int:
        length = len(nums)
        dp = [1] * length
        cnt = [1] * length
        ans = 1
        ans_cnt = 0
        for i in range(length):
            for j in range(0, i):
                if nums[i] > nums[j]:
                    if dp[j] + 1 > dp[i]:
                        cnt[i] = cnt[j]
                    elif dp[j] + 1 == dp[i]:
                        cnt[i] += cnt[j]
                    dp[i] = max(dp[i], dp[j] + 1)
                    ans = max(ans, dp[i])
        for i in range(length):
            if dp[i] == ans:
                    ans_cnt += cnt[i]
        return ans_cnt
```

- 题目：相交链表（leetcode-160）
- 题目描述：给你两个单链表的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null 。
- 思路：这道题显然利用双指针寻找相交节点。不妨设链表 A 表示为 a + c，链表 B 表示为 b + c，其中 c 是两链表的公共部分。我们可以使用两个指针分别遍历链表 A 和链表 B，当指针到达链表末尾时，重置指针到另一个链表的头部。这样，当两个指针相遇时，如果存在相交节点，那么它们会在相交节点处相遇；如果不存在相交节点，那么 c = null，那么它们会同时走完 a + b 的路程后到达 null。
- 时间复杂度：$O(m + n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> Optional[ListNode]:
        pa, pb = headA, headB
        while pa != pb:
            pa = pa.next if pa else headB
            pb = pb.next if pb else headA
        return pa
```

- 思路：这道题可以使用迭代或者递归的方法来反转
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        prev = None
        cur = head
        while cur:
            nxt = cur.next
            cur.next = prev
            prev = cur
            cur = nxt

        return prev
```

- 题目：回文链表（leetcode-234）
- 题目描述：给你一个单链表的头节点 head ，请你判断该链表是否为回文链表。如果是，返回 true ；否则，返回 false 。
- 思路：这题是一道双指针模板题，可以使用快慢指针先寻找链表中点，然后我们翻转后半段链表，和前半段比较即可。如果链表长度是奇数，我们找中点时跳过它，再往前走一步，因为它不参与比较（eg. 1->1->1->2->1->1->1）。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
class Solution:
    def isPalindrome(self, head: Optional[ListNode]) -> bool:
        if not head or not head.next:
            return True
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        if fast:
            slow = slow.next

        cur = slow
        prev = None
        while cur:
            nxt = cur.next
            cur.next = prev
            prev = cur 
            cur = nxt
        p1 = head
        p2 = prev
        while p2:
            if p2.val != p1.val:
                return False
            p1 = p1.next
            p2 = p2.next
        return True
```

- 题目：环形链表（leetcode-141）
- 题目描述：给你一个链表的头节点 head ，判断链表中是否有环。如果链表中存在环，则返回 true ；否则，返回 false 。
- 思路：用快慢指针来判断链表是否有环。快指针每次走两步，慢指针每次走一步，如果链表中存在环，那么快指针和慢指针最终会相遇；如果链表中不存在环，那么快指针会先到达链表末尾。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

            if slow == fast:
                return True
        return False
```

- 题目：环形链表 II（leetcode-142）
- 题目描述：给定一个链表的头节点 head ，返回链表开始入环的第一个节点。如果链表无环，则返回 null 。
- 思路：这题在判断链表是否有环的基础上，增加了寻找入环节点的步骤。首先，我们使用快慢指针来判断链表是否有环。如果链表中存在环，那么快指针和慢指针最终会相遇。此时，我们将其中一个指针重新指向链表的头节点，然后两个指针每次都走一步。当它们再次相遇时，相遇的节点就是链表开始入环的第一个节点。证明：设链表头节点到入环节点的距离为a，入环节点到相遇节点的距离为b，相遇节点到入环节点的距离为c，那么快指针走过的路程是慢指针的两倍，即2(a + b) = a + b + k * (b + c)，其中k是快指针在环内绕行的圈数。化简后得到a = (k - 1) * (b + c) + c，此时$a \equiv c \mod (b + c)$，因此当我们将其中一个指针重新指向链表的头节点，并且两个指针每次都走一步时，它们会在入环节点相遇。
- 时间复杂度：$O(n)$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

            if slow == fast:
                break
        if not fast or not fast.next:
            return None
        else:
            p = head
            while p != slow:
                p = p.next
                slow = slow.next
            return p
```

- 题目：合并两个有序链表（leetcode-21）
- 题目描述：将两个升序链表合并为一个新的 升序 链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。
- 思路：这题可以使用双指针实现，l1和l2分别指向两个链表的头节点，我们比较l1和l2当前指向的节点值，将较小的节点连接到新链表上，并将对应的指针向后移动一步。我们重复这个过程，直到其中一个链表被完全遍历。最后，我们将另一个链表剩余的部分连接到新链表上即可。
- 时间复杂度：$O(m + n)$。
- 代码实现：

```python
# Definition for singly-linked list.
from typing import Optional

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

class Solution:
    def mergeTwoLists(self, list1: Optional[ListNode], list2: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode()
        cur = dummy
        l, r = list1, list2

        while l and r:
            if l.val < r.val:
                cur.next = l
                l = l.next
            else:
                cur.next = r
                r = r.next
            cur = cur.next

        if l:
            cur.next = l
        if r:
            cur.next = r

        return dummy.next
```

- 题目：两数相加（leetcode-2）
- 题目描述：给你两个 非空 的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的，并且每个节点只能存储一位数字。请你将两个数相加，并以相同形式返回一个表示和的链表。你可以假设除了数字 0 之外，这两个数都不会以 0 开头。
- 思路：这题也是使用双指针来实现，我们分别用l1和l2指向两个链表的头节点，同时维护一个进位carry。我们比较l1和l2当前指向的节点值，将它们相加，并加上进位carry，得到当前位的和sum。然后，我们将sum对10取模得到当前位的值，并将sum除以10得到新的进位carry。我们将当前位的值连接到新链表上，并将对应的指针向后移动一步。我们重复这个过程，直到两个链表都被完全遍历，并且进位carry为0。最后，我们返回新链表的头节点即可。
- 时间复杂度：$O(\max(m, n))$。
- 代码实现：

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def addTwoNumbers(self, l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode()
        cur = dummy
        cnt = 0
        while l1 or l2 or cnt:
            x = l1.val if l1 else 0
            y = l2.val if l2 else 0

            tot = x + y + cnt
            cnt = tot // 10
            cur.next = ListNode(tot % 10)
            cur = cur.next
            if l1:
                l1 = l1.next
            if l2:
                l2 = l2.next
        return dummy.next
```
